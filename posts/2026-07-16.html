1. ChatGPT Voice: GPT-Live는 “대화형 에이전트”에 더 가깝다

이제 음성 모델이 기다렸다가 답하는 게 아니라, 들으면서 처리하고 뒤에서 다른 모델까지 부른다

OpenAI의 2026년 7월 8일 발표에 따르면 GPT-Live는 기존 음성 모드와 달리 full-duplex 구조를 사용해 사용자의 말을 들으면서 동시에 응답할 수 있다. 짧은 맞장구, 끊어 말하기 대응, 자연스러운 템포 유지가 핵심이다.

더 중요한 건 이 음성 모델이 혼자 답하지 않는다는 점이다. OpenAI는 복잡한 질문이 들어오면 GPT-Live가 뒤에서 최신 프런티어 모델에 작업을 위임한다고 설명했다. 출시 시점 기준으로는 GPT-5.5가 백그라운드 추론을 맡는다.

ChatGPT 릴리스 노트도 이를 확인한다. 유료 사용자는 GPT-Live-1, 무료 사용자는 GPT-Live-1 mini를 받으며, 웹과 iOS, Android에서 순차 배포 중이다. 시각 카드, 웹 검색, 메모리, 이미지 대화도 이어진다.

이번 변화의 핵심은 음성 품질 개선이 아니라, “말하는 인터페이스” 뒤에 실제 추론 엔진과 작업 위임 레이어를 붙였다는 점이다.

2. OpenAI: GPT-Red는 “AI가 AI를 깨뜨려 더 안전하게 만드는” 전략이다

성능 경쟁이 아니라 방어 자동화 경쟁이 본격적으로 보이기 시작했다

OpenAI의 2026년 7월 15일 안전 연구 공개에 따르면 GPT-Red는 내부 전용 자동 레드팀 모델이다. 목표는 배포 전 취약점을 대규모로 찾아내고, 그 공격을 다시 훈련 데이터로 써서 다음 모델을 더 견고하게 만드는 것이다.

OpenAI는 GPT-Red를 self-play reinforcement learning으로 훈련했다고 설명한다. 공격자 역할의 GPT-Red와 방어자 역할의 여러 모델을 동시에 학습시키며, 프롬프트 인젝션과 에이전트 오용 같은 시나리오를 반복적으로 파고든다.

수치도 세다. OpenAI는 GPT-5.6 Sol이 자사 가장 어려운 direct prompt injection 벤치마크에서 4개월 전 최고 운영 모델보다 실패가 6배 적었다고 밝혔다. 또 GPT-Red는 OpenAI 내부 실험에서 자판기 에이전트 가격을 바꾸고 주문을 취소시키는 식의 실제 공격 목표도 달성했다.

이 소식이 중요한 이유는 단순한 안전 PR이 아니기 때문이다. 이제 프런티어 랩은 사람 레드팀만으로는 부족하다고 보고, 모델 자체를 공격 자동화 도구로 쓰는 단계에 들어갔다.

3. Claude: 화려한 신기능보다 메모리와 규제 대응을 다듬고 있다

Anthropic은 지금 “더 많이 하게”보다 “더 안심하고 붙이게”에 집중하는 모습이다

Claude 공식 릴리스 노트에 따르면 7월 10일부터 Claude의 메모리는 하루 요약형에서 개별 카테고리 엔트리형으로 바뀌었다. Claude가 대화 중 메모리 항목을 읽고 갱신하는 구조로 옮겨가며, 이전보다 연속적인 개인화에 유리해졌다.

같은 릴리스 노트의 7월 14일 항목에서는 HIPAA readiness를 기업 관리자 스스로 설정할 수 있게 됐다고 밝힌다. Claude Enterprise와 Claude API 조직의 적격 관리자는 BAA 검토, 구현 가이드 다운로드, HIPAA 설정 활성화를 하나의 흐름에서 처리할 수 있다.

이 두 업데이트를 같이 보면 Anthropic의 최근 방향이 선명하다. Claude를 더 자주 쓰게 만드는 소비자형 자극보다, 기억의 품질과 민감 산업 도입 절차를 다듬어 실제 조직 도입 마찰을 줄이는 쪽이다.

4. Google: Gemini Notebook은 커졌지만, Gemini 3.5 Pro는 늦어지고 있다

기능 확장은 공격적이지만, 핵심 플래그십 모델 일정은 시장을 불안하게 만들었다

Google의 2026년 7월 16일 발표에 따르면 NotebookLM은 Gemini Notebook으로 이름이 바뀌었다. 단순 리브랜딩만이 아니라, Gemini 앱과 Google Search 안으로 더 깊게 들어가고, 각 노트북에 secure cloud computer를 붙여 코드를 직접 작성하고 실행할 수 있게 됐다.

Google은 이 기능이 복잡한 데이터 분석을 소스에 근거해 수행하게 해 준다고 설명했다. 즉 Notebook 계열 제품이 요약 도구에서 분석형 작업 공간으로 올라가고 있다.

하지만 같은 날 나온 Reuters 재전재 기사는 분위기를 뒤집었다. 보도에 따르면 Google은 Gemini 3.5 Pro 출시가 수개월 지연됐고, 특히 코딩 능력 개선 때문에 일정을 밀고 있는 것으로 전해졌다.

정리하면 Google의 오늘 뉴스는 이중적이다. 한쪽에서는 Gemini를 더 넓은 제품 표면에 밀어 넣고 있고, 다른 쪽에서는 가장 중요한 플래그십 모델 일정이 흔들리고 있다. 배포 확장 속도와 모델 완성도 사이의 긴장이 그대로 드러난 셈이다.
